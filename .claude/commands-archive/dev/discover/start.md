## Start a new coding session with AI

Start a new coding session, understand the overall project's goal, structure,
architecture, and progress name, and to pick up where we left off.

## Process
- **First, check current project state:** If `ai-docs/` directory exists, examine all files to understand what's already implemented:
  - `ai-docs/README.md` - Current project overview and goals
  - `ai-docs/ARCHITECTURE.md` - Current system design
  - `ai-docs/API.md` - Implemented external API endpoints and interfaces
  - `ai-docs/DEVELOPMENT.md` - Current development patterns
  - `ai-docs/PRD.md` - Completed requirements
  - `ai-docs/SCHEMA.md` - Current external database schemas and repository entities
  - `ai-docs/USAGE.md` - Current functionality
- Check for `CLAUDE.md` in project root for additional context
- Examine all of the `.md` files in the `/tasks/` directory for ongoing work.
- Examine any additional files referenced from them.
- DO NOT examine in files in the `/tasks/archive` directory.
- Write out a quick summary which includes:
  - Overall project goal (from `ai-docs/README.md` if available, otherwise from `/tasks/`)
  - What has been completed? (based on `ai-docs/` content and completed tasks)
  - What should our next task be? (based on incomplete tasks in `/tasks/`)
