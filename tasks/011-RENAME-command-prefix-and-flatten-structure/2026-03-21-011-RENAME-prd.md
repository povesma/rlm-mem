# 011-RENAME: Command Prefix Rename and Structure Flatten - PRD

**Status**: Draft
**Created**: 2026-03-21
**Author**: Claude (via rlm-mem analysis)

---

## Problem Statement

The current command prefix `rlm-mem` names the underlying technologies
(RLM + claude-mem), not the purpose of the tool. Users don't care
about the implementation stack — they care about their development
workflow. Additionally, the three-level command structure
(`/rlm-mem:discover:start`) is unnecessarily verbose; the middle tier
(`discover`, `plan`, `develop`, `support`) adds typing overhead without
user value.

**Who**: Developers using rlm-mem commands daily
**What**: Commands are verbose and named after implementation, not purpose
**Why**: Cognitive overhead on every invocation; the name doesn't explain
what the tool is; middle tier is invisible to users anyway
**Future context**: The rename sets up a cleaner foundation for a
planned profile/config system where users select workflow configurations
per project or task type (`/dev:profile`, `/dev:switch`)

---

## Goals

### Primary
- Rename command prefix from `rlm-mem` → `dev`
- Flatten three-level structure to two-level: `/dev:start`, `/dev:impl`
- Preserve the existing `dev` tree (currently deprecated) in an archive
  within the repo — not a separate branch

### Secondary
- All user-facing references (README, CLAUDE.md, command cross-references)
  updated consistently
- install.sh copies the new structure correctly
- Zero functional change — only naming/structure

## User Stories

### Story 1 — Flat command invocation

**As a** developer using Claude Code,
**I want** to type `/dev:impl` instead of `/rlm-mem:develop:impl`,
**so that** commands are shorter and easier to remember.

**Acceptance Criteria:**
- [ ] All 10 commands accessible under `/dev:` prefix
- [ ] No middle-tier segment in any command path
- [ ] Tab-completion in Claude Code works with new paths

### Story 2 — Old dev tree preserved

**As a** maintainer of this repo,
**I want** the deprecated `dev` command tree archived within the repo
(not deleted, not in a branch),
**so that** it's available for reference and not lost.

**Acceptance Criteria:**
- [ ] Old `dev` files moved to `.claude/commands-archive/dev/`
- [ ] Archive directory committed to the repo
- [ ] Not copied by install.sh (excluded from sync)

### Story 3 — Self-references updated

**As a** command file,
**I want** all cross-references to other commands updated to the new
`/dev:` prefix,
**so that** suggestions like "run `/rlm-mem:plan:prd`" don't break
after the rename.

**Acceptance Criteria:**
- [ ] All occurrences of `/rlm-mem:` replaced with `/dev:` in
  command files
- [ ] Phase path references (`develop/`, `plan/`, etc.) removed
  where they appear in cross-references

### Story 4 — Documentation updated

**As a** new user reading the README,
**I want** all examples to show the new `/dev:` commands,
**so that** I can copy-paste them without translation.

**Acceptance Criteria:**
- [ ] README Available Commands section updated
- [ ] README Quick Start examples updated
- [ ] README What Gets Installed tree updated
- [ ] CLAUDE.md file structure updated
- [ ] install.sh copies `dev/` not `rlm-mem/`

### Story 5 — install.sh updated

**As a** new installer,
**I want** `install.sh` to copy `.claude/commands/dev/` (not `rlm-mem/`),
**so that** the install reflects the new structure.

**Acceptance Criteria:**
- [ ] install.sh copies `commands/dev/` to `~/.claude/commands/dev/`
- [ ] install.sh does NOT copy `commands-archive/dev/`
- [ ] Running install.sh on a fresh system produces `/dev:*` commands

## Out of Scope

- **Renaming the repo** (`rlm-mem` repo name stays for now)
- **Renaming the skills** (`.claude/agents/` files that appear in
  skill lists — separate effort)
- **Profile/config system** — this rename is the prerequisite;
  profiles come after in a separate feature
- **Functional changes** — zero behavior change, rename only
- **The `rlm`, `coding`, `dev` skill trees** installed at user scope
  from other packages — not touched

---

## Risks

| Risk | Mitigation |
|------|------------|
| Users have existing `/rlm-mem:*` muscle memory | Document in README; old commands simply stop working after reinstall |
| `dev` conflicts with another command tree on some systems | install.sh overwrites — not a silent conflict |
| Cross-reference misses in command files | Grep for all `/rlm-mem:` occurrences before closing |
